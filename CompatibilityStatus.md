## Validation Policy

- Fast checks stay sequential: build, focused unit test classes, and selected smoke tests.
- Heavy compatibility media runs and multi-million-cycle true-drive serial-load milestones stay opt-in through environment variables and local untracked manifests under `C64/DISKS`.
- Phase 1 harness plumbing is effectively complete: local milestone runs can distinguish phase-filtered, pre-shard, post-shard, and final selected counts; aggregate summaries include selected milestone keys for dashboard/audit use, manifest validation failures are surfaced as summary acceptance failures, `SWIFT64_LOCAL_MILESTONE_LIMIT` caps post-shard execution, `SWIFT64_LOCAL_MILESTONE_MEDIA_LIMIT` only bounds media discovery, and `script/run_milestone_shards.sh` runs deterministic shards sequentially with a shared resumable JSONL log.
- `SWIFT64_LOCAL_MILESTONE_USEFUL_FEATURE_GATE=1` now enables a practical preservation-corpus acceptance preset for useful software coverage: it requires Phase 4-7 milestone presence, representative PRG/D64/G64/TAP/CRT media coverage, PAL/NTSC and C64/C64C profiles, compat/standard true-drive modes, 6581/8580 SID models, SID compatibility mode, explicit actions, max-cycle budgets, observable proof, explanatory expected-failure markers, and failure on unclassified or unexpected milestone outcomes. Explicit environment selections still merge into the preset instead of being replaced.
- Phase 2 CPU, memory, and bus certification is complete for the roadmap gate: focused unit slices cover core CPU behavior, interrupt/RDY/SO timing, C64 reset coupling, 6510 port overlay semantics, VIC-visible zero-page RAM under the port, ROM/I/O banking, color RAM/open-bus behavior, and VIC-driven RDY stalls. Public CPU corpus sweeps remain opt-in and can run as a single strict-cycle Tom Harte/SingleStepTests pass or deterministic sequential opcode shards through `script/run_cpu_json_shards.sh`; functional binary/manifest suites can likewise run through `script/run_cpu_functional_shards.sh`.
- ROMs, commercial media, protected images, screenshots, and generated binary artifacts must remain untracked unless explicitly approved.
